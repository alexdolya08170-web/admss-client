# Admss 

Веб-приложение для работы автодилера

## Возможности

- **Авторизация** — вход в систему, двухфакторная аутентификация (2FA), обновление токена доступа.
- **Главная панель** — обзор и быстрый доступ к разделам.
- **Инвентарь** — учёт автомобилей (список, создание и редактирование записей).
- **Контакты и сделки** — ведение контактов и сделок, в т.ч. операции washout по сделке.
- **Счета (accounts)** — работа со счетами, приём платежей (`take payment`).
- **Отчёты** — просмотр и настройка отчётов.
- **Экспорт на сайт** — планирование и управление выгрузкой на веб.
- **Задачи** — список и сводка по задачам.
- **Тест-драйв** — подготовка печатных форм для тест-драйва.
- **Профиль и настройки** — профиль пользователя, общие настройки, пользователи и роли.
- **Карты** — интеграция с Google Maps (при настройке API-ключа).

Доступ к разделам зависит от роли (например, часть функций недоступна роли `salesPerson`).

## Технологии

| Категория       | Стек |
|-----------------|------|
| UI              | React 18, TypeScript |
| Сборка          | Create React App (`react-scripts` 5) |
| Маршрутизация   | React Router v6 |
| Состояние       | MobX |
| HTTP-клиент     | Axios (интерцепторы, Bearer-токен) |
| Компоненты UI   | PrimeReact, PrimeIcons |
| Формы / валидация | React Hook Form, Formik, Yup |
| Прочее          | jsPDF (отчёты/PDF), WaveSurfer (аудио), react-grid-layout (сетки) |

Дополнительно: ESLint, Prettier, Husky, lint-staged.

## Требования

- **Node.js** (рекомендуется LTS) и **npm**
- Современный браузер (Chrome, Firefox, Safari — как в `browserslist` в `package.json`)

## Установка и запуск

```bash
npm install
npm start
```

Приложение откроется в режиме разработки (по умолчанию [http://localhost:3000](http://localhost:3000)).

Сборка production:

```bash
npm run build
```

Результат — каталог `build/`, готовый к раздаче статическим веб-сервером.

## Переменные окружения

Скопируйте `.env.example` в `.env` и при необходимости измените значения:

| Переменная | Назначение |
|------------|------------|
| `REACT_APP_API_URL` | Базовый URL API (по умолчанию `https://app.admss.com/api/v1/`) |
| `REACT_APP_VERSION` | Версия клиента для отображения/отладки |
| `REACT_APP_TYPE` | Тип приложения (например, `client`) |
| `REACT_APP_MAGIC` | Служебный идентификатор/ключ приложения |
| `REACT_APP_GOOGLE_MAPS_API_KEY` | Ключ Google Maps для карт |

Переменные с префиксом `REACT_APP_` встраиваются в сборку на этапе `npm start` / `npm run build`.

## Структура исходников (кратко)

- `src/index.tsx` — точка входа, маршруты приложения
- `src/App.tsx` — провайдер хранилища MobX и вложенные маршруты
- `src/dashboard/` — экраны панели (инвентарь, сделки, контакты и т.д.)
- `src/sign/` — вход и 2FA
- `src/http/` — Axios-инстансы, сервисы API, защищённые маршруты
- `src/store/` — MobX-сторы
- `src/common/` — константы, модели, провайдеры, хуки
