## Возможности

| Модуль | Описание |
|--------|----------|
| Авторизация | Вход в систему, 2FA (TOTP/SMS), refresh-токены, защита маршрутов |
| Главная панель | Дашборд с метриками, быстрые действия, уведомления |
| Инвентарь | Управление авто: список, фильтры, CRUD, загрузка фото, статусы |
| Контакты и сделки | CRM-функционал: лиды, клиенты, воронка продаж, операции washout |
| Счета (Accounts) | Выставление счетов, приём платежей (`take payment`), история транзакций |
| Отчёты | Гибкая настройка, экспорт в PDF/Excel, планирование рассылки |
| Экспорт на сайт | Синхронизация инвентаря с публичным сайтом, очереди задач |
| Задачи | Назначение, приоритеты, дедлайны, сводная аналитика |
| Тест-драйв | Генерация печатных форм, учёт автомобилей на тесте |
| Профиль и настройки | Управление профилем, ролями, правами доступа, системные настройки |
| Карты | Интеграция с Google Maps (геолокация дилеров, маршруты) |

## Технологический стек

```yaml
Frontend:
  UI: React 18 + TypeScript 5
  Сборка: Create React App (react-scripts 5)
  Маршрутизация: React Router v6 (data routers)
  Состояние: MobX 6 + MobX-React-Lite
  HTTP: Axios (интерцепторы, retry, Bearer-токен)
  Компоненты: PrimeReact 9 + PrimeIcons
  Формы: React Hook Form + Yup (валидация)
  Утилиты: 
    - jsPDF / react-pdf (отчёты)
    - WaveSurfer.js (аудиозаметки)
    - react-grid-layout (кастомизируемые дашборды)

DevOps & Quality:
  - ESLint + Prettier (единый стиль кода)
  - Husky + lint-staged (pre-commit хуки)
  - Jest + React Testing Library (тесты)
  - dotenv (управление конфигурацией)
```

## Установка и запуск

### Быстрый старт

```bash
# 1. Клонирование и установка зависимостей
git clone <repository-url>
cd admss-frontend
npm install

# 2. Настройка окружения
cp .env.example .env
# Отредактируйте .env при необходимости

# 3. Запуск в режиме разработки
npm start
```

Приложение будет доступно по адресу: http://localhost:3000

### Production-сборка

```bash
npm run build
# Результат: папка ./build/ — готова к деплою на Nginx, S3, Vercel и др.
```

## Переменные окружения

Создайте файл `.env` на основе `.env.example`:

| Переменная | Обязательна | Значение по умолчанию | Описание |
|------------|-------------|----------------------|----------|
| `REACT_APP_API_URL` | Да | `https://app.admss.com/api/v1/` | Базовый URL бэкенд-API |
| `REACT_APP_VERSION` | Нет | `1.0.0` | Версия клиента (отображается в UI) |
| `REACT_APP_TYPE` | Нет | `client` | Тип сборки (`client` / `admin` / `demo`) |
| `REACT_APP_MAGIC` | Да | — | Служебный ключ приложения |
| `REACT_APP_GOOGLE_MAPS_API_KEY` | Нет | — | Ключ для интеграции с Google Maps |
| `REACT_APP_ENABLE_MOCK_API` | Нет | `false` | Режим моков для разработки |

## Роли и доступ

Доступ к функциям контролируется через систему ролей:

| Роль | Инвентарь | Сделки | Отчёты | Настройки | Экспорт |
|------|-----------|--------|--------|-----------|---------|
| admin | Полный | Полный | Полный | Полный | Полный |
| manager | Полный | Полный | Чтение | Чтение | Полный |
| salesPerson | Чтение | Свои | Свои | Нет | Нет |
| accountant | Чтение | Чтение | Фин.отчёты | Нет | Нет |

Обозначения:
- Полный — полный доступ на чтение и запись
- Чтение — только просмотр
- Свои — доступ только к своим записям
- Нет — доступ запрещён

Роли настраиваются в `src/common/constants.ts` и проверяются через хук `useRole()`.

Учётные данные для тестирования в среде разработки:

```
Логин: mobile5
Пароль: password
```

> Внимание: данные учётные данные предназначены только для локальной разработки и тестирования. Не используйте их в продакшене.

Для входа:
1. Запустите приложение: `npm start`
2. Перейдите по адресу: http://localhost:3000
3. Введите логин и пароль из таблицы выше
4. При включённой 2FA используйте тестовый код из настроек или отключите проверку в `.env`

## Безопасность

### Рекомендации

- Все запросы к API используют `Authorization: Bearer <token>`
- Токены хранятся в памяти приложения или в `httpOnly` cookie (при поддержке бэкенда)
- Реализован автоматический рефреш токена при получении ошибки 401
- Все формы проходят валидацию на клиенте и сервере
- В продакшене отключите `console.log` и source maps для чувствительных модулей

### Настройка для разработки

```bash
# Включить режим моков (если поддерживается)
REACT_APP_ENABLE_MOCK_API=true npm start

# Отключить проверку 2FA локально
# (требует поддержки на стороне бэкенда)
```

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| `npm start` не запускается | Убедитесь, что порт 3000 свободен: `lsof -i :3000` |
| Ошибка подключения к API | Проверьте `REACT_APP_API_URL` и доступность бэкенда |
| Карты не загружаются | Добавьте валидный `REACT_APP_GOOGLE_MAPS_API_KEY` |
| Сборка падает с ошибкой TypeScript | Запустите `npx tsc --noEmit` для детального вывода |
| Не применяются стили PrimeReact | Убедитесь, что темы импортированы в `index.tsx` |

### Логи и отладка

```bash
# Включить подробные логи Axios
localStorage.setItem('debug', 'axios:*')

# Просмотреть версию приложения
// В консоли браузера:
console.log(process.env.REACT_APP_VERSION)
```
